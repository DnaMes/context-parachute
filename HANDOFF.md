# HANDOFF — context-parachute

## Goal

Ship SemVer versioning for context-parachute: a canonical version, a changelog,
git tags, and version provenance stamped into every generated handoff artifact.

## Current Progress

Done and released as **v1.0.0** (committed `5e19785`, tagged `v1.0.0`, pushed to
`origin/main` + tag on GitHub `DnaMes/context-parachute`).

- `VERSION` file at repo root = single source of truth (`1.0.0`).
- `CHANGELOG.md` in Keep a Changelog format, v1.0.0 backfilled from git log.
- Both hooks read `VERSION` fail-open (missing/unreadable → `unknown`, never errors)
  and stamp the producing version into the watcher directive, the emergency
  snapshot, and (via the skill) every artifact footer.
- `install.sh` prints the version on run.
- `tests/run-tests.sh` asserts `VERSION == CHANGELOG top == latest git tag`.
- `README.md` documents the 3-step release process.

## What Worked

- Fail-open version read via `VERSION` file next to the script (`BASH_SOURCE`-relative).
- Consistency test caught nothing broken but guards future drift cheaply.
- Live-verified stamping: watcher emits `v1.0.0`, emergency.md header carries it.

## What Didn't Work / Rejected

- **Version in `~/.claude/parachute.json`** — rejected. Config is user-owned and the
  installer never clobbers it, so a stamped version there goes stale on upgrade.
  Version describes the code, not the user's settings.

## Decisions Made

- Stay at **1.0.0** (not 0.x): shipped README/commits already say "v1"; retconning
  would contradict shipped docs.
- Hooks *do* read the version at runtime — justified by artifact provenance (trace a
  wrong-looking handoff to the version that wrote it), with a fail-open guard.
