# context-parachute — v1.0.1 + v1.1.0 Design Spec

**Date:** 2026-07-09
**Status:** Approved design, pre-implementation
**Author:** brainstormed with Andre
**Base:** v1.0.0 (`5e19785`, tagged `v1.0.0`)

## Summary

Two releases, split by intent:

- **v1.0.1 — bugfix.** Fix a confirmed live misfire on 1M-context sessions, plus a
  portability fix so the watcher runs on macOS/BSD. Both are small, both correct,
  both touch only `parachute-watch.sh`.
- **v1.1.0 — feature.** Re-fire the parachute after an actual compaction, so a long
  session that fills → compacts → refills gets a second handoff instead of going
  silent for the rest of its life.

## v1.0.1 — bugfix release

### The bug (confirmed, not hypothetical)

The watcher computes `percent = observed_tokens / context_window` with
`context_window` defaulting to `200000`. On an Opus 4.8 `[1m]` session the real
window is 1,000,000 tokens.

Evidence (this session, 2026-07-09): observed token total reached **305,157** —
already past the 200k default — and the threshold that fires the parachute is
80% × 200,000 = **160,000 absolute tokens ≈ 16% of the real 1M window.** Two
`.fired` markers were written today at that point. So **every Opus-1M session
parachutes at ~16% fill.** That is a correctness bug, not missing polish.

### Why not auto-detect the window (rejected approach)

Two auto-detection ideas were considered and rejected:

1. **Model-string → window mapping.** The transcript's `.message.model` reads
   `claude-opus-4-8` with **no `[1m]` suffix**, even on a 1M session. The model
   string does not carry the window. Probed and confirmed.
2. **Empirical: infer window from observed tokens** (`max(config, observed)` or
   "if observed > 200k, snap to 1M"). This is **circular** and fails at exactly
   the threshold: on a fresh 1M session the false fire happens at 160k tokens —
   *before* any observation proves the window is larger than 200k. You need the
   window at 160k; you get evidence of it only after 200k. Cannot work.

A whole-transcript grep for a structural window signal (`contextWindow`,
`1000000`, `max_tokens`) returned only false positives — matches that came from
this session's own tool output, not from any CC-emitted structural field. No
reliable window signal exists in the transcript as of CC 2.1.205.

### The fix: per-project `context_window` override (B2, honest form)

The config machinery already exists — the watcher already reads `context_window`
from global `~/.claude/parachute.json` and per-project `.parachute/config.json`
(`parachute-watch.sh` lines 40–56). The fix is **documentation + defaults**, not
new code:

- Keep `context_window` default at `200000` (correct for the common Sonnet/Opus
  non-1M case).
- Document, in README and in the config comment, that **1M-context sessions must
  set `context_window: 1000000`** — globally if the user runs 1M by default, or
  per-project via `.parachute/config.json`.
- Add a `.parachute/config.json` example to the README showing the 1M override.

No watcher logic changes for B2. The knob is honest: the user knows their window;
the tool cannot reliably detect it, so it exposes the value instead of guessing.

**Optional guard (include):** if `context_window` is still the 200000 default AND
observed tokens already exceed it, emit a one-line `WARN:` to stderr — "observed
tokens exceed configured context_window; if this is a 1M session set
context_window: 1000000". Fail-open, never blocks. This turns a silent misfire
into a visible, self-documenting nudge. It does NOT change the trigger decision.

### C2 — macOS/BSD portability

`parachute-watch.sh:77` uses `tac`, which is GNU-only (absent on macOS/BSD; and
`tail -r`, the BSD reverse, is absent on GNU — so a reverse-read approach needs
two code paths). Avoid reverse-read entirely:

```sh
# before
USAGE_LINE="$(tac "$TRANSCRIPT" | head -n 500 | jq '…' | head -n 1)"
# after — take the LAST match instead of reversing to take the first
USAGE_LINE="$(tail -n 500 "$TRANSCRIPT" | jq '…' | tail -n 1)"
```

`tail` is POSIX-portable. Identical window (last 500 lines) and identical result
(last matching assistant-usage line). One-line change, one code path.

### v1.0.1 acceptance

- Existing 34 tests still pass (the token-sum fixtures feed the watcher the same
  way; `tail`-based read must produce identical results — verify against
  `normal.jsonl` → 80%, `cache-heavy.jsonl` → 79%, `corrupt.jsonl` → 85%).
- New test: config override — a `.parachute/config.json` with

- Optional-guard test: default window + observed > default → WARN on stderr,

## v1.1.0 — feature release: re-fire after compaction (A3)

### The gap

The fired-marker is one-shot per `session_id`
(`$TMPDIR/context-parachute/<session_id>.fired`). A session that reaches 80%,
parachutes, then gets auto-compacted down and refills to 80% again **never fires
a second time** — the marker is still present. Long sessions get exactly one
handoff, at the first threshold crossing, and then go silent.

### The fix

Reset (delete) the fired-marker after an **actual compaction**, so the next
threshold crossing arms the parachute again.

- **Trigger on `PostCompact` only.** Not SessionStart: a new session already has a
  new `session_id` (marker absent → reset is a no-op), and on a *resumed* session
  a SessionStart reset risks an immediate spurious re-fire. `PostCompact` is the
  precise signal — a compaction just happened, so re-arm.
- New hook `parachute-postcompact.sh` (or extend behaviour), registered by the
  installer as a `PostCompact` hook alongside the existing `PreCompact` one.
- Logic: read `session_id` from stdin JSON, `rm -f` the marker for that session.
  Fail-open: no session_id / no marker / any error → exit 0, silent.

### ⚠️ Verify-before-build (blocks implementation)

`PostCompact` stdin fields are **not yet confirmed**. The existing
`~/bin/claude-hook-post-compact.sh` reads no stdin at all (uses `pwd`), so it is
no evidence of what CC sends. **Before writing `parachute-postcompact.sh`, dump
the raw PostCompact stdin from a real compaction and confirm `session_id` is
present.** If PostCompact does not carry `session_id`:

- Fallback A: key the marker by `cwd` (project path) instead of session_id — then
  PostCompact can reset by cwd. But cwd-keying weakens the one-shot guarantee
  across concurrent sessions in the same repo; evaluate only if session_id is
  absent.
- Fallback B: reset ALL markers older than N minutes on any PostCompact. Blunt,
  but PostCompact is rare enough that collateral re-arming is acceptable.

Pick the fallback only after the probe. Do not build A3 on an assumed field.

### v1.1.0 acceptance

- New test: marker present → PostCompact with matching session_id → marker gone →
  watcher fires again on next threshold crossing.
- PostCompact with no session_id / no marker → silent exit 0 (fail-open).
- Installer registers the PostCompact hook idempotently; uninstall removes it;
  existing PostCompact entries preserved (mirror the existing installer tests).
- shellcheck clean. VERSION → `1.1.0` (SemVer minor: a backward-compatible
  feature, no breaking change), CHANGELOG, tag `v1.1.0`.

## Out of scope (unchanged from v1)

- Per-agent tailored prompts (`continue-codex.md`) — AGENTS.md covers Codex/OpenCode.
- Continuous per-turn state (Sonovore-style) — token cost every turn.
- Autonomous-run trigger via PostToolUse (B1) — deferred to a later release; the
  PreCompact fallback still covers the no-prompt case.

## Decisions log (this spec)

| Decision | Choice | Why |
|---|---|---|
| 1M window detection | config override, not auto-detect | model string lacks `[1m]`; empirical inference is circular at the threshold; no structural transcript signal |
| Misfire severity | v1.0.1 bugfix, not a v1.1.0 feature | confirmed live: fires at ~16% on every Opus-1M session |
| Portable reverse-read | `tail -n 500 … \| tail -1` | `tac` GNU-only, `tail -r` BSD-only; last-match needs neither |
| Re-fire trigger | PostCompact only | new session → new id (no-op); resumed session → SessionStart risks spurious fire |
| A3 field dependency | probe PostCompact stdin first | existing local hook reads no stdin — session_id presence unconfirmed |
